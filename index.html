1. Introduction

This style guide is our visual identity. Within it you'll find an outline of how DoSomething.org communicates our brand visually. Our style guide is a living, breathing document that will evolve over time and change as we change.

This style guide is not our entire brand. Our brand is far more than what we show. For non-visual brand guidelines please see our Communications Guide.

A few points: our style guide lives in code and is hosted in the cloud. That means that at any time the most up-to-date version of it will always be accessible to you. Perhaps more importantly, that means that the style guide you see here is the actual HTML and CSS that will be used to build our site. Any changes made here will be seen across our entire web platform. A PDF version will also be made accessible in the near future.

View this style guide and its included guidelines as a set of tools. If you find yourself being called upon to do any design work please consult this style guide before starting. It will guide you through the design process. Feedback on design work should be justified and passed with reference to the relevant section of the style guide. If you have any suggestions or find it to be at all incomplete please add an issue to our Trello board.

2. Design Tenents

These tenents are the foundation of design at DoSomething.org. They serve both as a framework for creation and critique.

Clean

First and foremost, Do Something’s visual identity is clean. This is not a license to be boring.* Simple, uncluttered design provides a strong foundation from which we can build at scale.

Purposeful

All decisions made should be justifiable. Do Something’s visual brand is clean and consistent. We do not clutter and we do not decorate without reason.

Youthful

This is who we are. More importantly, that is who our users are. Our visual identity should reflect this.

3. Color

Color is to be used consistently and purposefully. We have a four-tone greyscale complimented by a call to action color and two accent colors.

  • Blue

  • Our blue is a color used primarily for interactable elements on web platforms. It is the default call to action color and should be used for primary links, buttons, navigation items, etc. This blue should not be used as an accent or background color. Headers and other typographic elements should not be set in this blue unless they are linked elements.

  •  

  • Purple

  • Purple is used to call attention to significant points of user interaction (e.g. the "report back" section of the campaign action page) or typographic elements to break up the monotony of long passages of type. Uses: second-level headers, background color of system messages and important content regions containing actionable elements.

  •  

  • Yellow

  • Yellow is used as a skeuomorphic accent to all handwritten elements in our applications. All text set using the Engineer typeface should be accompanied by an organic highlight mark under the text. Yellow should not be used for slides, page elements or call to action elements (e.g. buttons, etc.).

  •  

  • Greyscale

  • Our greyscale is used primarily to color block background elements. These page elements are typically header, navigation, aside or footer elements. By setting a darker background on these elements we are able to clearly differentiate these page regions, group relevant content and diminish the visual importance of secondary or tertiary page sections (e.g. a footer). The lightest of greys is useful when an accent or border is needed to break up content set against a white background.

4. Typography

Content on the web is consumed primarily in written form. For this reason it's incredibly important that our typography be clear and consistent across all public-facing communication channels.

Font Families

Proxima Nova is our base typeface. It is a clean, modern sans serif typeface designed by Mark Simonson. In addition, a cursive font named L'Engineer is available for use.

Proxima Nova is the only typeface other than Engineer that is approved for use. No other font is permitted for use on any public-facing asset, slide deck, web site, banner advertisement, etc.

Engineer is to be used sparingly to provide additional information and call attention to important aspects of a content region. On the web site it is to be used exclusively with a yellow highlight mark.

Font Sizes

Font Weights

Applying various font weights helps to create a separation between content and establish a hierarchy on the page. We use Proxima Nova in three weights: bold, semi-bold and regular. Generally, headers should be set in either the bold or semi-bold weight depending on the size and body copy should be set using the regular weight. When in doubt, start heavier and drop down as your font size decreases.

Engineer is only available in a regular font weight.

Text Width

Text width is an important aspect of legibility. The width of all text regions should be restricted based on the context and font size in question. Text should be no wider than roughly 60 characters for text blocks set using a 16px font size with the exception of longform documents (e.g. a cease and desist letter) may be set with standard 1" margins.

Text Color

All text, with the exception of secondary headers (purple) and interactable elements (blue), should be set using an off-black. Our text color is:

  • HEX: #222222
  • RGB: rgb(35, 35, 35)
